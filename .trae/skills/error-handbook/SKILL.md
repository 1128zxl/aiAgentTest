# 错误处理手册 Skill

本 Skill 用于管理和查询项目中的错误处理记录。

## 功能说明

### 1. 查询已知错误
当用户遇到错误时：
1. 读取 `docs/error-handbook.md` 查找是否有记录
2. 如果找到，输出解决方案
3. 如果没找到，帮助用户排查并记录

### 2. 添加新错误
当用户解决了新错误时：
1. 询问错误描述、错误信息、解决方案
2. 询问是否需要错误截图
3. 更新 `docs/error-handbook.md`
4. 可选：生成 HTML 格式报告

### 3. 生成报告
当用户要求生成错误报告时：
1. 读取 Markdown 格式的手册
2. 转换为 HTML 格式
3. 保存到 `docs/error-report-{编号}.html`

## 文档结构

```
docs/
├── error-handbook.md           # 主手册（Markdown）
├── error-report-001-xxx.html   # 单个错误的 HTML 报告
└── error-report-002-xxx.html   # ...
```

## 错误记录模板

```markdown
### 错误 #{编号}: {简短标题}

**日期**: {日期}
**严重程度**: 🔴 高 / 🟡 中 / 🟢 低
**状态**: ✅ 已解决 / ⏳ 待解决

**问题描述**:
{详细描述}

**错误信息**:
```
{错误代码/错误日志}
```

**根本原因**:
{分析}

**解决方案**:
{解决方案}

**验证方法**:
{验证步骤}
```

## 使用示例

### 查询错误
```
用户: Docker Desktop 打不开了
AI: 让我查一下错误手册...
    找到了！错误 #001: Docker Desktop 重装后无法启动
    解决方案：执行 wsl --shutdown && netsh winsock reset
```

### 添加新错误
```
用户: 解决了 npm install 报错的问题
AI: 好的，请提供：
    1. 错误信息是什么？
    2. 错误代码或截图？
    3. 解决方案是什么？
```

## 规则

1. 每个错误分配唯一编号（从 001 开始递增）
2. 错误报告文件名格式：`error-report-{编号}-{简短标题}.html`
3. 严重程度定义：
   - 🔴 高：阻塞主要功能，无法继续工作
   - 🟡 中：部分功能受影响
   - 🟢 低：非关键问题
4. 保持文档更新，添加新错误后更新主手册
